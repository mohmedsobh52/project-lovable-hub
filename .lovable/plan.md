

# خطة بناء نظام إدارة المشاريع (PMS)

## تحليل الملف المرفوع

الملف `PMS.txt` يحتوي على صفحة Index واحدة بحجم 2263 سطر تعتمد على **30+ مكون مخصص** و**10+ hooks** و**Supabase backend functions** غير موجودة في المشروع الحالي. لا يمكن نسخ الملف مباشرة لأن جميع التبعيات مفقودة.

## النهج المقترح

بناء النظام على مراحل، بدءاً من الهيكل الأساسي ثم إضافة الميزات تدريجياً.

---

## المرحلة 1: الهيكل الأساسي والتنقل

- **Layout مع Sidebar** يحتوي على أقسام النظام الرئيسية
- **صفحات أساسية**: لوحة التحكم، المشاريع، تحليل BOQ، المقاولين، الإعدادات
- **دعم RTL واللغة العربية** مع hook للغة (`useLanguage`)
- **تصميم احترافي** بنمط glass-card والألوان المتدرجة

## المرحلة 2: لوحة التحكم الرئيسية (Dashboard)

- **KPI Cards**: إجمالي المشاريع، القيمة الإجمالية، البنود، حالة المشاريع
- **رسوم بيانية**: توزيع التصنيفات، مخطط القيم
- **قائمة المشاريع الأخيرة** مع حالة كل مشروع

## المرحلة 3: رفع وتحليل BOQ

- **مكون FileUpload**: رفع PDF و Excel
- **استخراج النصوص**: معالجة محلية للـ Excel (بدون AI)
- **عرض النتائج**: جدول البنود مع التصنيفات والأسعار والكميات
- **WorkflowStatus**: شريط تقدم المراحل (رفع ← استخراج ← تحليل ← WBS ← تصدير)

## المرحلة 4: أدوات متقدمة

- **مقارنة BOQ** بين إصدارات مختلفة
- **إدارة المقاولين** وربطهم بالبنود
- **تصدير P6** للجدولة
- **المرفقات** وإدارة الملفات
- **الإشعارات** وتذكيرات العقود
- **قوالب BOQ** جاهزة

## المرحلة 5: المصادقة والـ Backend (اختياري)

- ربط Supabase للمصادقة وحفظ المشاريع
- حفظ/تحميل المشاريع من قاعدة البيانات
- تحليل AI عبر Supabase Edge Functions

## التفاصيل التقنية

### الملفات الرئيسية التي سيتم إنشاؤها

```text
src/
├── hooks/
│   └── useLanguage.ts          # دعم العربي/الإنجليزي + RTL
├── components/
│   ├── AppSidebar.tsx           # القائمة الجانبية
│   ├── FileUpload.tsx           # رفع الملفات
│   ├── WorkflowStatus.tsx       # شريط تقدم المراحل
│   ├── AnalysisResults.tsx      # عرض نتائج التحليل
│   ├── KPIDashboard.tsx         # بطاقات KPI
│   ├── BOQComparison.tsx        # مقارنة BOQ
│   ├── SubcontractorManagement.tsx
│   ├── P6Export.tsx
│   ├── ProjectAttachments.tsx
│   └── NotificationSettings.tsx
├── lib/
│   ├── excel-utils.ts           # معالجة Excel
│   └── local-excel-analysis.ts  # تحليل محلي
├── pages/
│   ├── Index.tsx                # لوحة التحكم
│   ├── Analysis.tsx             # صفحة التحليل
│   ├── Projects.tsx             # المشاريع
│   └── Settings.tsx             # الإعدادات
```

### الأولوية

سأبدأ بالمراحل 1-3 (الهيكل + Dashboard + تحليل BOQ) لأنها تشكل 80% من القيمة الوظيفية. المراحل 4-5 تُضاف لاحقاً حسب الحاجة.

### ملاحظة مهمة

الملف الأصلي يعتمد بشكل كبير على Supabase Edge Functions للتحليل بالـ AI. في المرحلة الأولى سأبني التحليل المحلي (بدون AI) الذي يعمل مباشرة في المتصفح، ثم يمكن إضافة الـ AI لاحقاً.
